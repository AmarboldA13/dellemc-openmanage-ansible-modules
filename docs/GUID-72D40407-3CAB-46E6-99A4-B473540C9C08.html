Configure Quick Deploy settings

Module: ome_device_power_settings
Synopsis

This module allows to configure the Quick Deploy settings of a server or IOM on OpenManage Enterprise Modular.

Options

Table 1. ome_device_power_settings
Parameter Required Default Choices Comments
hostname True NA NA OpenManage Enterprise Modular IP address or hostname
username True NA NA OpenManage Enterprise Modular username
password True NA NA OpenManage Enterprise Modular password
port False NA NA OpenManage Enterprise Modular HTTPS port
validate_certs False True True, False
  • If C(False), the SSL certificates will not be validated.
  • Configure C(False) only on personally controlled sites where self-signed certificates are used.
  • Prior to collection version C(5.0.0), the I(validate_certs) is C(False) by default.
ca_path False NA NA
  • The Privacy Enhanced Mail (PEM) formatted file that contains a CA certificate to be used for the validation.
  • I(ca_path) is required if I(validate_certs) is C(True).
timeout False 30 NA The socket level timeout in seconds.
device_id False NA NA
  • The ID of the chassis for which the Quick Deploy settings to be deployed.
  • If the device ID is not specified, this module updates the Quick Deploy settings for the I(hostname).
  • I(device_id) is mutually exclusive with I(device_service_tag).
device_service_tag False NA NA
  • The service tag of the chassis for which the Quick Deploy setting needs to be updated.
  • If the device service tag is not specified, this module updates the Quick Deploy settings for the I(hostname).
  • I(device_service_tag) is mutually exclusive with I(device_id).
setting_type True NA [ServerQuickDeploy, IOMQuickDeploy]
  • The type of the Quick Deploy settings to be applied.
  • - C(ServerQuickDeploy) to apply the server Quick Deploy settings.
  • - C(IOMQuickDeploy) to apply the IOM Quick Deploy settings.
job_wait False True NA Determines whether to wait for the job completion or not.
job_wait_timeout False 120 NA
  • The maximum wait time of I(job_wait) in seconds. The job is tracked only for this duration.
  • This option is applicable when I(job_wait) is C(True).
quick_deploy_options True NA NA The Quick Deploy settings for server and IOM quick deploy.

Sub options:

  • Password:
    • The password to login to the server or IOM.
    • The module will always report change when I(password) option is added.
  • ipv4_enabled: Enables or disables the IPv4 network.
  • ipv4_network_type:
    • Choices: Static, DHCP
    • IPv4 network type.
    • I(ipv4_network_type) is required if I(ipv4_enabled) is C(True).
    • C(Static) to configure the static IP settings.
    • C(DHCP) to configure the Dynamic IP settings.
  • ipv4_subnet_mask:
    • IPv4 subnet mask.
    • I(ipv4_subnet_mask) is required if I(ipv4_network_type) is C(Static).
  • ipv4_gateway:
    • IPv4 gateway.
    • I(ipv4_gateway) is required if I(ipv4_network_type) is C(Static).
  • ipv6_enabled: Enables or disables the IPv6 network.
  • ipv6_network_type:
    • Choices: Static, DHCP
    • IPv6 network type.
    • I(ipv6_network_type) is required if I(ipv6_enabled) is C(True).
    • C(Static) to configure the static IP settings.
    • C(DHCP) to configure the Dynamic IP settings.
  • ipv6_prefix_length:
    • IPV6 prefix length.
    • I(ipv6_prefix_length) is required if I(ipv6_network_type) is C(Static).
  • ipv6_gateway:
    • IPv6 gateway.
    • I(ipv6_gateway) is required if I(ipv6_network_type) is C(Static).
  • Slots: The slot configuration for the server or IOM.

    Suboptions:

    • slot_id:
      • Required: True
      • The ID of the slot.
    • slot_ipv4_address: The IPv4 address of the slot.
    • slot_ipv6_address: The IPv6 address of the slot.
    • vlan_id: The ID of the VLAN.
