Configure server interface profile

Module: ome_server_interface_profile
Synopsis

This module allows to configure server interface profiles on OpenManage Enterprise Modular.

Options

Table 1. ome_server_interface_profiles
Parameter Required Default Choices Comments
hostname True NA NA OpenManage Enterprise Modular IP address or hostname
username True NA NA OpenManage Enterprise Modular username
password True NA NA OpenManage Enterprise Modular password
port False NA NA OpenManage Enterprise Modular HTTPS port
validate_certs False True True, False
  • If C(False), the SSL certificates will not be validated.
  • Configure C(False) only on personally controlled sites where self-signed certificates are used.
  • Prior to collection version C(5.0.0), the I(validate_certs) is C(False) by default.
ca_path False NA NA
  • The Privacy Enhanced Mail (PEM) formatted file that contains a CA certificate to be used for the validation.
  • I(ca_path) is required if I(validate_certs) is C(True).
timeout False 30 NA The socket level timeout in seconds.
device_id False NA NA
  • Device id of the Server under chassis fabric.
  • I(device_id) and I(device_service_tag) is mutually exclusive.
device_service_tag False NA NA
  • Service tag of the Server under chassis fabric.
  • I(device_service_tag) and I(device_id) is mutually exclusive.
nic_teaming False NA LACP, NoTeaming, Other
  • NIC teaming options.
  • C(NoTeaming) the NICs are not bonded and provide no load balancing or redundancy.
  • C(LACP) use LACP for NIC teaming.
  • C(Other) use other technology for NIC teaming.
nic_configuration False NA NA NIC configuration for the Servers to be applied.
Suboptions:
  • nic_identifier:
    • ID of the NIC or port number.
    • C(Note) This will not be validated.
    • Required: True
  • team:
    • Group two or more ports. The ports must be connected to the same pair of Ethernet switches.
    • I(team) is applicable only if I(nic_teaming) is C(LACP).
  • untagged_network:
    • The maximum or minimum VLAN id of the network to be untagged.
    • The I(untagged_network) can be retrieved using the M(ome_network_vlan_info)
    • If I(untagged_network) needs to be unset this needs to be sent as C(0)
    • C(Note) The network cannot be added as a untagged network if it is already assigned to a tagged network.
  • tagged_networks:
    • List of tagged networks.
    • Network cannot be added as a tagged network if it is already assigned to untagged network.
    • state:
      • Indicates if a list of networks needs to be added or deleted.
      • C(present) to add the network to the tagged list.
      • C(absent) to delete the Network from the tagged list.
      • Choices: present, absent
      • Default: present
    • names:
      • List of network name to be marked as tagged networks.
      • The I(names) can be retrieved using the M(ome_network_vlan_info)
      • Required: True
job_wait False True NA Provides the option to wait for job completion.
job_wait_timeout False 120 NA
  • The maximum wait time of I(job_wait) in seconds. The job is tracked only for this duration.
  • This option is applicable when I(job_wait) is C(True).

Return Values 

msg:
  description: Status of the overall server interface operation.
  returned: always
  type: str
  sample: Successfully triggered apply server profiles job.
job_id:
  description: Job ID of the task to apply the server interface profiles.
  returned: on applying the Interface profiles
  type: int
  sample: 14123

  description: Details of the HTTP Error.

Examples 
name: Modify Server Interface Profile for the server using the service tag
  dellemc.openmanage.ome_server_interface_profiles:
    hostname: "192.168.0.1"
    username: "username"
    password: "password"
    ca_path: "/path/to/ca_cert.pem"
    device_service_tag:
      - SVCTAG1
      - SVCTAG2
    nic_teaming: LACP
    nic_configuration:
      - nic_identifier: NIC.Mezzanine.1A-1-1
        team: no
        untagged_network: 2
        tagged_networks:
          names:
            - vlan1
      - nic_identifier: NIC.Mezzanine.1A-2-1
        team: yes
        untagged_network: 3
        tagged_networks:
          names:
            - range120-125

- name: Modify Server Interface Profile for the server using the device id
  dellemc.openmanage.ome_server_interface_profiles:
    hostname: "192.168.0.1"
    username: "username"
    password: "password"
    ca_path: "/path/to/ca_cert.pem"
    device_id:
      - 34523
      - 48999
    nic_teaming: NoTeaming
    nic_configuration:
      - nic_identifier: NIC.Mezzanine.1A-1-1
        team: no
        untagged_network: 2
        tagged_networks:
          names:
            - vlan2
      - nic_identifier: NIC.Mezzanine.1A-2-1
        team: yes
        untagged_network: 3
        tagged_networks:
          names:
            - range120-125
